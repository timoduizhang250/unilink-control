## 一、版本目标

1.4.15 是一次基础收口版本，不加入新的产品大功能。目标是让 Windows、macOS、Android 使用同一版本规则，避免旧安装、旧配置和错误连接线路互相污染，并建立一次可恢复的三端发布流程。

当前基线：

- Windows 正在运行 `C:\Program Files\UniLink Control` 中的 `1.4.14+72`；旧 AppData `1.4.11+69` 仍存在，但当前未运行。
- Mac 正在运行 `/Applications/UniLink Control.app` 中的 `1.4.12+70`，局域网端口 `21118` 可达。
- GitHub 当前最新公开版本为 `1.4.13`。
- Android 当前未连接 ADB；开始实施时必须先读取实际安装版本。

## 二、明确延期

根据 2026-07-17 的产品决定，下列问题不纳入 1.4.15：

- [ ] 不解决 UniLink 或 TigerVNC 的既有黑屏、首帧超时、编码器回退和显示器/锁屏诊断问题。
- [ ] 不修复 Mac 的 USB Wi-Fi、系统网络、DNS 或无法访问 GitHub 的问题。
- [ ] 不扩展 Mac 窗口模式，不做首页或设置的大规模 UI 改版。
- [ ] 不新增免费服务器线路，不承诺解决远距离公共中继的基础延迟。

边界规则：

- 既有黑屏可以作为已知问题延期，但 1.4.15 新改动导致的新增黑屏或连接回归仍然阻止发布。
- Mac 在线自动更新不作为 1.4.15 验收项；Mac 安装包仍必须构建、校验，并通过局域网手动安装验证。
- 不得把“端口可达”或“安装成功”写成“远程控制全部稳定”。

## 五、P0-C 连接意图、线路隔离与有限重连

- [ ] 用明确连接意图区分：UniLink ID 公网、局域网直连、自定义服务器、原生 VNC/RDP。
- [ ] IPv4、IPv6、局域网主机名和 `.local` 目标走直接路径，不继承 WebSocket、代理或隐式强制中继。
- [ ] 官方线路、自定义线路和局域网之间切换时，不修改无关设备的直连偏好。
- [ ] 技术性临时中继不得写回永久的 `force-always-relay` 设备设置。
- [ ] 原始 IP 或主机名直连失败时不静默切换公网，只提供明确的回退选择。
- [ ] 对 `10054`、`Reset by peer`、短暂服务重启和网线抖动执行有限次数重连；达到上限后停止并显示原因。
- [ ] 会话和日志显示实际路径：设备直连、局域网直连或公网中继。
- [ ] 官方线路与局域网往返切换 5 次，`192.168.137.2` 每次仍保持局域网直连配置。

验收：连接线路不会互相污染；失败后不会无限转圈，也不会把局域网目标错误送入公共服务器。
