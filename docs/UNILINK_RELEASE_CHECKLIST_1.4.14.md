# UniLink Control 1.4.14 更新执行清单

Updated: 2026-07-16
Release theme: Windows 安装一致性、局域网连接稳定性、黑屏与更新恢复

## 版本边界

1.4.14 是基础可靠性版本，不加入新的首页设计、Mac 窗口模式扩展或新的免费服务器线路。所有 P0 项完成前不得发布。

## 当前已知事实

- [x] Windows 存在两套安装：Program Files 1.4.12 服务/托盘/主界面，以及 AppData 1.4.11 旧用户安装和旧快捷方式。
- [x] Windows 到 Mac 的 `192.168.137.2:21118` 可达，但失败日志出现会话启动后快速退出或 `Reset by the peer`。
- [x] Mac 连接档案曾保存 `force-always-relay = 'Y'`，全局官方线路又会保存 `allow-websocket = 'Y'`，导致局域网 IP 被公网中继策略污染。
- [~] `src/client.rs` 已加入直接 IP 绕过隐式中继/WebSocket/代理的保护；2 个针对性 Rust 测试通过，Windows 1.4.14 Release 安装包也已生成并完成静态版本/DLL 哈希核对，但尚未安装到当前混装电脑，也未完成真机连接验收。
- [ ] TigerVNC 当前黑屏根因尚未确认。2026-07-12 曾成功进入同一台 Mac，因此不能把“端口 5900 可达”等同于画面可用。
- [ ] 当前 Program Files 版本与 Mac 的连续连接、输入和重连尚未重新验收。

## P0-1 Windows 单一安装

- [ ] 安装前盘点 Program Files、AppData、服务、托盘、桌面快捷方式、开始菜单、开机启动和协议注册的实际路径与版本。
- [ ] 备份旧安装中的设备身份、密码配置、连接记录和用户设置；迁移成功前不删除旧数据。
- [ ] 让 1.4.14 安装器识别旧 AppData 安装并执行可恢复迁移。
- [ ] GUI、托盘、服务、更新器、快捷方式和协议启动全部指向 `C:\Program Files\UniLink Control`。
- [ ] 安装完成后禁止旧 AppData 程序被快捷方式、开机启动或命令行连接重新唤起。
- [ ] 卸载和再次安装保持幂等，不遗留第二套活动程序。

验收：重启 Windows 后只运行一套 UniLink；GUI、托盘、服务的版本和核心文件哈希一致；旧配置和设备 ID 保留。

## P0-2 局域网与公网路由隔离

- [~] 直接 IPv4/IPv6 地址不继承 `force-always-relay`、WebSocket 或代理产生的隐式强制中继；针对性单元测试已通过。
- [ ] 用明确的连接意图区分“局域网直连、官方公网、自定义服务器、原生 VNC/RDP”，不要只靠地址字符串猜测。
- [ ] 支持局域网主机名和 `.local` 地址走直接路径，不能只覆盖纯 IP。
- [ ] 区分“用户主动要求始终中继”和“当前线路技术上需要中继”，后者不得写回永久的设备偏好。
- [ ] 官方线路、HITOHA/自定义线路和局域网之间切换时，不修改无关设备的直连偏好。
- [ ] 原始 IP 直连失败时不得静默改走公网；只有设备存在有效 ID 且公网可用时才提供清晰的回退选项。
- [ ] 会话界面显示真实连接方式：设备直连、局域网直连或公网中继。

验收：官方线路与局域网之间往返切换 5 次，`192.168.137.2` 每次仍走直连；不再自动写回 `force-always-relay = 'Y'`。

## P0-3 UniLink 黑屏与断线

- [ ] 连接前检查 Mac UniLink 服务、屏幕录制权限、辅助功能权限和有效显示器，不只检查端口。
- [ ] 首帧设置超时并分类报告：权限缺失、捕获为 `0x0`、编码器失败、网络断开或对端重置。
- [ ] 硬件编码或解码没有产出首帧时，自动回退软件路径一次，并记录实际使用的编码器。
- [ ] 防止 Mac `--server` 重复实例、端口占用和启动循环再次出现。
- [ ] 对短暂网线断开或服务重启做有限次数重连，不能无限转圈或无提示退出。

验收：Windows 到 Mac 直连后 10 秒内出现画面，稳定运行 10 分钟；鼠标、拖动、键盘和剪贴板可用；主动断开和重新连接连续 10 次通过。

## P0-4 TigerVNC 黑屏

- [ ] 区分 TigerVNC 外部客户端黑屏和 UniLink 自身视频黑屏，两条路径使用独立状态和日志。
- [ ] VNC 探测不只验证 TCP 5900，要验证 RFB 握手和认证后的首个有效帧。
- [ ] 检查 Mac 屏幕共享/远程管理服务、当前登录会话、显示器睡眠与锁屏状态。
- [ ] 记录 TigerVNC 启动目标、退出码和可用日志；失败时给出中文原因，不只显示“已打开客户端”。
- [ ] 验证 TigerVNC 1.16.0 的编码、色深和共享会话参数，必要时使用一组已验证的兼容参数。
- [ ] TigerVNC 失败不得改变 UniLink 的局域网/公网路由配置。

验收：从 UniLink 打开 TigerVNC 后 10 秒内看到 Mac 当前桌面，鼠标和键盘可用；关闭后再次连接 5 次均无黑屏。
